## Why

DRO-12 задает базовый каркас нового приложения: структуру репозитория, технологический стек, контрактный workflow, локальное окружение и начальную доменную модель. Без этого команда не имеет единого способа запускать фронт, бекенд, PostgreSQL, Swagger UI и Prisma Studio, а будущие endpoint-ы рискуют расходиться между контрактом, реализацией и потреблением на фронте.

## What Changes

- Добавить проектную структуру `/backend`, `/frontend`, `/docs`, `docker-compose.yml`, `.env.example` и `README.md` для приложения по PRD-01.
- Зафиксировать стек: Next.js 14 API routes only + Prisma + PostgreSQL 15 для бекенда, React 18 + Vite + Tailwind CSS для фронтенда.
- Ввести контрактный workflow: новый endpoint сначала описывается в `docs/openapi.yaml`, затем реализуется в бекенде и только после этого потребляется фронтендом.
- Настроить единый формат API-ответов `{ data, error, meta }`, размещение API-роутов в `/backend/app/api/` и Swagger UI на `http://localhost:3001/api/docs`.
- Настроить независимый локальный запуск: PostgreSQL через `docker-compose up`, бекенд на порту `3001`, фронтенд на порту `3000`.
- Добавить централизованную конфигурацию через `.env` и шаблон `.env.example` без значений по умолчанию; фронтенд должен использовать `VITE_API_URL`.
- Добавить Prisma-схему для стран, городов, направлений, университетов, требований поступления, программ, пользователей, избранного и лидов.
- Добавить seed-данные: 5-10 стран, 10-20 городов, 6-8 направлений, 20-30 университетов, 50-100 программ, 1 тестовый пользователь и 3-5 лидов.
- Обновить README инструкциями локального запуска, проверки Swagger UI и просмотра данных через `npx prisma studio`.

## Capabilities

### New Capabilities

- `project-scaffold`: Репозиторий SHALL содержать согласованную структуру приложения PRD-01, базовые package scripts и документацию локального запуска.
- `runtime-configuration`: Проект SHALL загружать runtime-настройки только из `.env` и поддерживать актуальный `.env.example` без секретов и значений по умолчанию.
- `api-contract-workflow`: API SHALL развиваться contract-first через `docs/openapi.yaml`, единый формат ответов и Swagger UI на бекенде.
- `domain-data-model`: Бекенд SHALL содержать Prisma-схему PostgreSQL с доменными сущностями, связями и seed-данными для локальной разработки.

### Modified Capabilities

Пока нет существующих capabilities для изменения.

## Impact

- Новые каталоги и файлы: `backend/`, `frontend/`, `docs/openapi.yaml`, `docker-compose.yml`, `.env.example`, `README.md`.
- Новые Node.js зависимости в подпроектах: Next.js 14, React 18, Vite, Tailwind CSS, Prisma, Swagger UI-интеграция и PostgreSQL client.
- Новая инфраструктура локальной разработки: PostgreSQL 15 в Docker с volume для данных.
- Новые команды проверки и запуска для Node.js подпроектов, плюс обязательные существующие проверки Go-репозитория `go fmt ./...` и `go test ./...`.
- Потенциальное пересечение с текущим Go-оркестратором ограничивается корнем репозитория, README, `.env.example` и общими правилами конфигурации; существующее поведение CLI не должно ломаться.
