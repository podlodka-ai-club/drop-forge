## Why

Оркестратору нужен базовый модуль интеграции с Linear, чтобы кодинг-агенты могли читать задачи, переводить их между статусами и работать с комментариями без ручного переключения в UI. Сейчас в проекте нет ни клиента Linear, ни согласованного подхода к описанию модулей, поэтому дальнейшее развитие интеграций быстро приведет к разрозненным решениям.

## What Changes

- Добавить новый модуль Linear для чтения задач по идентификатору с получением названия и описания.
- Возвращать название текущего столбца задачи, так как логика оркестратора будет привязываться к столбцам.
- Добавить операции смены статуса задачи в Linear через целевой workflow state/column.
- Добавить чтение комментариев задачи и публикацию новых комментариев.
- Добавить минимальную CLI-команду для получения списка задач только с Linear-доски, заданной через ENV, как временную точку ручной проверки модуля.
- Добавить централизованную конфигурацию Linear через `.env` и `.env.example`.
- Зафиксировать верхнеуровневый формат документации модулей, чтобы у каждого модуля была краткая карточка с назначением, зависимостями, точками входа и сценариями использования.

## Capabilities

### New Capabilities
- `linear-issues`: модуль Linear для чтения issue, смены статуса, чтения комментариев и публикации комментариев.
- `module-docs`: единый формат верхнеуровневой документации модулей и место хранения таких описаний в репозитории.

### Modified Capabilities
- None.

## Impact

- Код: новый пакет интеграции с Linear, обновление загрузки конфигурации, возможное подключение модуля в CLI/оркестратор.
- Конфигурация: новые переменные окружения для доступа к Linear и выбора доски для списка задач, включая `LINEAR_BOARD_ID`.
- Документация: новая структура модульных описаний и первичная документация для Linear-модуля.
- Внешние зависимости: HTTP API Linear и схема workflow/state в рабочем пространстве.
