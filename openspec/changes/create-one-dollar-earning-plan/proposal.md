## Why

Linear-задача DRO-29 сформулирована как "Нужен план чтобы заработать $1", но в репозитории нет явного артефакта, который превращает такой результат в конкретный, проверяемый план действий. Нужен минимальный документ, который фиксирует путь к первому платному результату без изменения кода оркестратора и без преждевременного усложнения продукта.

## What Changes

- Добавить документационный артефакт с практическим планом заработка первого `$1` на базе текущих возможностей проекта.
- В плане зафиксировать целевое предложение, минимальный способ получения оплаты, шаги исполнения, критерии успеха и ограничения.
- Сохранить план как обычный markdown-документ в `docs/`, чтобы его можно было ревьюить и итеративно обновлять.
- Не менять runtime-поведение CLI, конфигурацию, Linear-интеграцию или proposal-runner flow.

## Capabilities

### New Capabilities

- `one-dollar-earning-plan`: Репозиторий SHALL содержать проверяемый markdown-план заработка первого `$1`, связанный с текущим назначением `orchv3` и пригодный для ручного исполнения.

### Modified Capabilities

Пока нет существующих capabilities для изменения.

## Impact

- Новый документ `docs/one-dollar-earning-plan.md`.
- Новая OpenSpec capability `one-dollar-earning-plan`.
- Без новых runtime-зависимостей.
- Без изменений публичного поведения CLI/API и без изменения `.env.example`.
