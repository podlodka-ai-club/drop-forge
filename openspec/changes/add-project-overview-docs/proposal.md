## Why

Текущий `README.md` помогает запустить проект, но не отвечает коротко и связно на вопрос "что это за проект и как он устроен". Для новых участников и внешних reviewers не хватает отдельного обзорного описания на русском языке, которое быстро объясняет назначение `orchv3`, его ключевые акторы и proposal-stage workflow.

## What Changes

- Добавить отдельную обзорную capability с человеко-ориентированным описанием проекта, его цели и основного orchestration flow.
- Зафиксировать требования к содержанию обзорной документации: назначение проекта, ключевые компоненты, жизненный цикл Linear-задачи и ссылки на детальные документы.
- Обновить требования к `README.md`, чтобы корневой entrypoint явно ссылался на новый обзорный документ и помогал перейти от краткого запуска к архитектурному контексту.

## Capabilities

### New Capabilities
- `project-overview-docs`: обзорная документация, которая объясняет назначение проекта, основные компоненты и поток proposal-stage без погружения в реализацию.

### Modified Capabilities
- `project-readme`: `README.md` должен направлять читателя к обзорной документации проекта для быстрого понимания контекста и терминов.

## Impact

- `README.md`
- новый обзорный документ в `docs/`
- OpenSpec specs для документации проекта
