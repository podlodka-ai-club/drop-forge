## ADDED Requirements

### Requirement: Система читает задачу Linear по идентификатору
Система SHALL получать задачу Linear только по пользовательскому identifier вида `ENG-123` и возвращать как минимум поля `title` и `description`, достаточные для работы оркестратора с содержимым задачи.

#### Scenario: Успешное чтение задачи по identifier
- **WHEN** вызывающий код запрашивает задачу по identifier вида `ENG-123`
- **THEN** система возвращает задачу с identifier, названием, описанием и текущим state

#### Scenario: Internal ID не принимается как входной идентификатор
- **WHEN** вызывающий код передает internal ID Linear вместо identifier вида `ENG-123`
- **THEN** система возвращает ошибку валидации и не отправляет запрос в Linear

#### Scenario: Задача не найдена
- **WHEN** вызывающий код запрашивает несуществующую задачу
- **THEN** система возвращает явную доменную ошибку "issue not found"

### Requirement: Система получает список задач Linear
Система SHALL получать список задач Linear только с доски, заданной конфигурацией, и возвращать для каждой задачи identifier, название, описание и текущий state.

#### Scenario: Успешное получение списка задач
- **WHEN** CLI-команда запрашивает список задач Linear и в конфигурации задана доска
- **THEN** система выводит список задач с identifier, state и названием каждой задачи

#### Scenario: Доска для списка задач не задана
- **WHEN** CLI-команда запрашивает список задач Linear без настроенной доски
- **THEN** система возвращает ошибку конфигурации и не отправляет запрос в Linear

### Requirement: Система переводит задачу в целевой workflow state
Система SHALL уметь менять state задачи Linear по identifier задачи и целевому state reference, где reference может быть ID состояния или его именем в workflow команды задачи.

#### Scenario: Перевод задачи по имени столбца
- **WHEN** вызывающий код передает identifier задачи и имя целевого state
- **THEN** система находит state в workflow команды задачи и переводит задачу в него

#### Scenario: Целевой state неизвестен
- **WHEN** вызывающий код передает имя state, которого нет в workflow команды задачи
- **THEN** система не выполняет обновление и возвращает ошибку валидации целевого state

### Requirement: Система читает комментарии задачи
Система SHALL возвращать комментарии задачи Linear в хронологическом порядке с телом комментария как обычным текстом, автором и временем создания.

#### Scenario: У задачи есть комментарии
- **WHEN** вызывающий код запрашивает комментарии существующей задачи
- **THEN** система возвращает непустой список комментариев, отсортированный по времени создания от старых к новым

#### Scenario: У задачи нет комментариев
- **WHEN** вызывающий код запрашивает комментарии задачи без комментариев
- **THEN** система возвращает пустой список без ошибки

#### Scenario: Rich text не нормализуется
- **WHEN** Linear возвращает комментарий с форматированием
- **THEN** система не строит markdown или rich text модель и возвращает только обычное текстовое тело комментария

### Requirement: Система публикует комментарий в задачу
Система SHALL создавать новый комментарий в указанной задаче Linear и возвращать данные созданного комментария.

#### Scenario: Успешная публикация комментария
- **WHEN** вызывающий код передает существующую задачу и непустой текст комментария
- **THEN** система создает комментарий в Linear и возвращает его идентификатор, автора и время создания

#### Scenario: Пустой комментарий отклоняется до вызова API
- **WHEN** вызывающий код передает пустое тело комментария
- **THEN** система возвращает ошибку валидации и не отправляет запрос в Linear
