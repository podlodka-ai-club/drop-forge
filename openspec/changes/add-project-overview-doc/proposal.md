## Why

Текущий репозиторий уже содержит `README.md`, `architecture.md` и несколько узких документов в `docs/`, но у них разная глубина и целевая аудитория. Для запроса уровня "Расскажи о проекте" нет одного короткого обзорного документа, который быстро объясняет назначение `orchv3`, его текущий scope, ключевые компоненты и границы системы без погружения в детали реализации.

## What Changes

- Добавить отдельный обзорный документ проекта в `docs/` с объяснением назначения `orchv3`, текущего proposal-stage workflow, основных внутренних акторов и ограничений текущей итерации.
- Зафиксировать в обзорном документе, как связаны `README.md`, `architecture.md`, `docs/proposal-runner.md` и `docs/linear-task-manager.md`, чтобы читатель понимал, где искать следующий уровень детализации.
- Обновить корневой `README.md`, чтобы он явно ссылался на новый обзорный документ как на рекомендуемую следующую точку чтения после быстрого входа в проект.
- Не менять CLI, конфигурацию, orchestration flow и runtime-контракты; изменение ограничено репозиторной документацией.

## Capabilities

### New Capabilities
- `project-overview-docs`: Репозиторий SHALL содержать отдельный обзорный документ проекта с кратким описанием назначения, архитектурных ролей, текущих workflow и ограничений системы.

### Modified Capabilities
- `project-readme`: README SHALL направлять читателя к обзорному документу проекта как к следующему уровню документации после базового entrypoint-описания.

## Impact

- Новый файл документации в `docs/`, ориентированный на обзор проекта, а не на отдельный subsystems walkthrough.
- Обновление `README.md` в части навигации по документации.
- Использование существующих артефактов `README.md`, `architecture.md`, `docs/proposal-runner.md` и `docs/linear-task-manager.md` как источников правды.
- Без новых runtime-зависимостей, без изменений `.env.example` и без изменения поведения CLI.
