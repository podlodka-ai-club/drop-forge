## Why

Сейчас `orchv3` трактует любой непустой текст как задачу на создание OpenSpec proposal и запускает внешний workflow с `git`, `codex` и `gh`. Из-за этого простой запрос вроде `Расскажи мне о проекте` приводит к ненужному proposal-runner сценарию вместо локального ответа по текущему репозиторию.

## What Changes

- Добавить в CLI отдельный режим обзорного ответа о проекте для информационных запросов о текущем репозитории.
- Определять, что входной текст относится к обзору проекта, до запуска proposal workflow.
- Формировать ответ локально из проверяемых источников репозитория, чтобы не требовать временного клона, GitHub-доступа и внешних CLI.
- Сохранить существующий proposal workflow для запросов, которые действительно описывают изменение или новую задачу.
- Добавить покрытие тестами для выбора режима, формируемого обзора и отсутствия внешних команд в обзорном режиме.

## Capabilities

### New Capabilities
- `project-overview-mode`: CLI предоставляет краткий обзор проекта по информационному запросу о репозитории без запуска proposal workflow.

### Modified Capabilities
Пока нет.

## Impact

- Изменения в `cmd/orchv3/main.go` и связанном потоке обработки пользовательского ввода.
- Новый или выделенный модуль, который собирает обзор проекта из локальных артефактов репозитория.
- Изменения в тестах CLI и proposal runner для подтверждения выбора корректного режима.
- Без новых внешних runtime-зависимостей; обзорный режим должен работать без `git`, `codex`, `gh` и без сетевого доступа.
