## Why

Сейчас ошибки в консоли выглядят так же, как информационные события, поэтому при запуске оркестратора вручную важные сбои легко пропустить в потоке JSON-логов. Нужно сделать ошибки визуально заметными, не ломая существующий машинно-читаемый формат логов.

## What Changes

- Добавить подсветку консольных логов с `type=error` красным цветом при выводе в интерактивный терминал.
- Сохранить неизменный JSON Lines формат для неинтерактивного вывода, тестов, пайпов и внешних log sink-ов.
- Не менять структуру события, набор полей, значения `type` и текст ошибок.
- Не добавлять новых внешних runtime-зависимостей, если подсветку можно реализовать стандартной библиотекой Go.

## Capabilities

### New Capabilities

- Нет.

### Modified Capabilities

- `structured-logging`: добавляется требование к человеко-читаемому консольному отображению error-событий без изменения машинного JSON-контракта.

## Impact

- Затрагивает настройку CLI-логгера и/или writer-обертку вокруг stderr/stdout для интерактивной консоли.
- Может потребоваться небольшой helper в `internal/steplog` или `cmd/orchv3`, чтобы централизованно применять ANSI-коды к error-событиям.
- Тесты должны покрыть, что error-события подсвечиваются в терминальном режиме, а обычный writer продолжает получать валидные JSON Lines без ANSI-кодов.
- `.env.example` не меняется, если подсветка включается автоматически по интерактивному терминалу и не добавляет новый runtime-параметр.
