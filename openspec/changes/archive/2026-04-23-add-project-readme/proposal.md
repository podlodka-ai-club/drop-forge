## Why

Сейчас в репозитории нет корневого README, поэтому новый участник проекта не видит быстрый способ понять назначение `orchv3`, его зависимости, переменные окружения и фактический сценарий запуска. Это уже мешает использовать CLI и повышает риск расхождения между кодом, `.env.example` и отдельной документацией в `docs/`.

## What Changes

- Добавить корневой `README.md` с кратким описанием назначения проекта и текущего сценария использования CLI.
- Задокументировать минимальные шаги локального запуска, включая запуск с аргументом CLI и через `stdin`.
- Перечислить runtime- и tool-зависимости проекта: Go, `git`, `codex`, `gh`, `.env` и требуемую аутентификацию для GitHub.
- Описать ключевые переменные окружения и сослаться на `.env.example` как на шаблон доступной конфигурации.
- Собрать в одном месте практическую информацию, которая сейчас размазана между кодом и `docs/proposal-runner.md`.

## Capabilities

### New Capabilities

- `project-readme`: Репозиторий SHALL содержать корневой README с актуальным описанием назначения CLI, способов запуска, внешних зависимостей и конфигурации окружения.

### Modified Capabilities

Пока нет существующих capabilities для изменения.

## Impact

- Новый файл `README.md` в корне репозитория.
- Возможное уточнение или переиспользование информации из `docs/proposal-runner.md`, чтобы избежать дублирования и расхождений.
- Использование существующих артефактов проекта как источника правды: `cmd/orchv3/main.go`, `internal/config/config.go`, `.env.example`, `go.mod`.
- Без новых runtime-зависимостей и без изменения поведения CLI.
