## Why

В проекте уже появились несколько однотипных stage-runner'ов (`proposalrunner`, `applyrunner`, `archiverunner`) и близкие тестовые наборы вокруг них. Перед дальнейшим ростом оркестратора нужно проверить фактические дубли, отделить допустимую симметрию стадий от вредного копирования и предложить точечные исправления без преждевременного усложнения архитектуры.

## What Changes

- Добавить явный аудит дублей по Go-коду и тестам с классификацией повторов по риску и выгоде исправления.
- Зафиксировать рекомендации по исправлению в отдельном документе, чтобы команда могла видеть, какие дубли стоит устранять сейчас, а какие лучше оставить до подтвержденной потребности.
- Выполнить только низкорисковые исправления, если аудит выявит механические дубли без stage-specific различий, например общие helper'ы для запуска команд, Codex executor plumbing или тестовые builders.
- Сохранить публичное поведение CLI, runtime-конфигурацию, Linear transitions, git workflow и OpenSpec workflow без изменений.
- Не добавлять внешние анализаторы дублей как обязательную зависимость проекта; аудит должен опираться на стандартные Go-инструменты и ручную проверку, если отдельный инструмент не будет явно обоснован.

## Capabilities

### New Capabilities

- `code-duplicate-audit`: описывает обязательный результат аудита дублей, критерии классификации и правила применения безопасных исправлений.

### Modified Capabilities

- Нет.

## Impact

- `docs/duplicate-code-audit.md`: новый отчет с найденными дублями, классификацией и предложенными исправлениями.
- `internal/proposalrunner`, `internal/applyrunner`, `internal/archiverunner`: основные кандидаты на проверку и возможное извлечение механически общих частей.
- `internal/reviewrunner` и `internal/coreorch`: вторичный scope для проверки повторяющихся orchestration/test patterns.
- Тесты затронутых пакетов: обновление или добавление покрытия для любых выполненных рефакторингов.
- `architecture.md`: обновлять только если исправления реально меняют границы ответственности или вводят новый общий пакет.
