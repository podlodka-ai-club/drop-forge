## Why

У приложения `DROP-FORGE` нет зафиксированного фирменного логотипа, поэтому проекту не хватает визуальной идентичности для README, GitHub, документации и будущих интерфейсов. Сейчас стоит оформить единый узнаваемый знак, пока бренд еще не размыт случайными временными изображениями.

## What Changes

- Создать официальный логотип `DROP-FORGE` с согласованной визуальной концепцией, отражающей тему капли, металла и ковки.
- Подготовить минимальный бренд-пакет: основной lockup, иконку без текста, монохромные и инверсные варианты.
- Добавить экспортируемые файлы в векторном и растровом форматах для использования в документации, репозитории и интерфейсах.
- Задокументировать базовые правила использования логотипа: допустимые фоны, минимальный размер, отступы и запрещенные трансформации.

## Capabilities

### New Capabilities
- `drop-forge-logo`: Репозиторий содержит официальный пакет логотипа `DROP-FORGE` с вариантами ассетов и краткими правилами использования.

### Modified Capabilities

Пока нет существующих capabilities для изменения.

## Impact

- Новый каталог бренд-ассетов в репозитории, например `docs/branding/drop-forge-logo/`.
- Новые файлы логотипа в форматах `SVG` и `PNG`, плюс краткий usage guide.
- Возможное обновление существующей документации или `README`, чтобы использовать официальный логотип вместо временных материалов.
- Без новых runtime-зависимостей и без изменения текущего поведения Go CLI.
