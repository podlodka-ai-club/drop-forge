## Why

Нужен автоматизированный сценарий подготовки OpenSpec-предложения для внешнего GitHub-проекта: оператор передает описание задачи, а приложение само клонирует репозиторий, запускает Codex CLI с нужным skill, публикует изменения и возвращает ссылку на PR. Это сокращает ручные шаги и делает работу агента наблюдаемой через консольные логи.

## What Changes

- Добавить модуль запуска задачи, который принимает строку с описанием задачи.
- Читать runtime-настройки из `.env`, включая URL GitHub-репозитория и параметры для создания PR.
- Поддержать шаблон переменных в `.env.example` как единственный источник правды по доступным ENV без секретов и значений по умолчанию.
- Создавать временную рабочую директорию и клонировать туда проект через `git clone`.
- Запускать Codex CLI в клонированном проекте со skill `openspec-propose` и промптом из описания задачи.
- Логировать в консоль этапы выполнения: создание temp-директории, git clone, отправленный prompt, поток вывода Codex CLI, создание ветки/PR и итоговую ссылку.
- После генерации OpenSpec-артефактов создавать pull request с изменениями в целевом GitHub-репозитории и возвращать URL PR вызывающему коду.
- Покрыть новую логику тестами с подменяемыми исполнителями команд и файловой системой там, где это практически необходимо.

## Capabilities

### New Capabilities

- `codex-proposal-pr-runner`: Сценарий запуска OpenSpec-предложения через Codex CLI для GitHub-проекта с последующим созданием PR и наблюдаемыми логами.

### Modified Capabilities

Пока нет существующих capabilities для изменения.

## Impact

- Новый внутренний пакет для orchestration flow: конфигурация, временная директория, git clone, запуск Codex CLI, создание PR.
- Обновление `cmd/orchv3` для вызова сценария из CLI или подготовки публичного API модуля.
- Расширение `internal/config` для чтения `.env` и новых переменных окружения.
- Обновление `.env.example` без секретов и значений по умолчанию.
- Новая Go-зависимость `github.com/joho/godotenv` для загрузки `.env`.
- Возможная зависимость от установленных внешних CLI: `git`, `codex`, и инструмент для GitHub PR (`gh`) или прямой GitHub API-клиент из стандартной библиотеки.
