## Why

Для локальной разработки и тестирования приложению нужен предсказуемый способ поднять Elasticsearch без ручной установки и настройки на машине разработчика. Docker-контейнер решает эту задачу единообразно и снижает порог запуска окружения.

## What Changes

- Добавить Docker-конфигурацию для локального запуска Elasticsearch рядом с приложением.
- Зафиксировать минимальные runtime-настройки Elasticsearch для dev-сценария: single-node режим, сетевой порт, volume для данных и healthcheck.
- Добавить переменные окружения приложения для подключения к локальному Elasticsearch и синхронизировать `.env.example`.
- Описать команды запуска, остановки и проверки доступности Elasticsearch в документации проекта.
- Уточнить ожидаемый поток локальной разработки: как поднять сервис, как приложению подключаться к нему и что делать при первом запуске.

## Capabilities

### New Capabilities

- `docker-elasticsearch-runtime`: локальный runtime-сценарий, в котором Elasticsearch запускается через Docker с документированными настройками и точкой подключения для приложения.

### Modified Capabilities

Пока нет существующих capabilities для изменения.

## Impact

- Новый Docker-манифест в корне репозитория, например `compose.yaml` или `docker-compose.yml`.
- Обновление `.env.example` и кода конфигурации для Elasticsearch-переменных подключения.
- Документация по локальному запуску и диагностике Elasticsearch.
- Возможное добавление именованного Docker volume для хранения данных между перезапусками.
