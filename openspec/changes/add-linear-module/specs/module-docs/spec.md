## ADDED Requirements

### Requirement: Репозиторий хранит верхнеуровневую документацию модулей в едином каталоге
Репозиторий SHALL хранить верхнеуровневую документацию модулей в каталоге `docs/modules`, где каждому модулю соответствует отдельный документ, а индекс каталога содержит список доступных модулей.

#### Scenario: Добавление нового модуля
- **WHEN** в проекте появляется новый модуль с публичным поведением или внешней интеграцией
- **THEN** в `docs/modules` создается документ модуля и ссылка на него добавляется в индекс

### Requirement: Документ модуля имеет фиксированную структуру
Каждый документ модуля SHALL содержать разделы "Назначение", "Границы ответственности", "Публичные операции", "Зависимости", "Конфигурация", "Отказные сценарии и ограничения" и "Связанные изменения".

#### Scenario: Создание документа для Linear-модуля
- **WHEN** команда документирует новый Linear-модуль
- **THEN** документ содержит все обязательные разделы и описывает операции списка задач с доски из `LINEAR_BOARD_ID`, чтения задачи по identifier, смены state и работы с комментариями

### Requirement: Документация модуля обновляется вместе с публичными изменениями
Система SHALL требовать обновления документа модуля в рамках того же change, если меняются публичные операции модуля, его конфигурация или внешние зависимости.

#### Scenario: Изменение конфигурации модуля
- **WHEN** модуль получает новую обязательную переменную окружения
- **THEN** change включает обновление документа модуля и отражает новую настройку в разделе конфигурации
