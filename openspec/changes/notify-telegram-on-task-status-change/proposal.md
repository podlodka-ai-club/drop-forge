## Why

Сейчас изменение статуса задачи в Linear выполняется как локальная операция task manager без расширяемого механизма реакции на этот факт. Для DRO-23 нужно отправлять уведомление в Telegram при смене статуса задачи и сделать это через event-driven подход, чтобы новые реакции добавлялись подписчиками на события, а не изменением вызывающего кода.

## What Changes

- Добавить доменное событие успешного изменения статуса задачи, публикуемое после подтвержденного перехода в Linear.
- Добавить простой in-process event bus/dispatcher для подписчиков на события task manager.
- Добавить Telegram-уведомитель как подписчик на событие изменения статуса задачи.
- Добавить runtime-конфигурацию Telegram bot token и chat ID через `.env`/`.env.example`.
- Покрыть сценарии публикации события, доставки уведомления и обработки ошибок тестами.
- Не менять публичный сценарий вызова `MoveTask`: вызывающий код продолжает запрашивать смену статуса, а реакции выполняются через события.

## Capabilities

### New Capabilities

- `task-status-notifications`: event-driven уведомления о смене статуса задачи, включая Telegram-подписчик и правила доставки.

### Modified Capabilities

- `linear-task-manager`: успешный переход статуса управляемой задачи публикует событие изменения статуса с данными задачи и целевого состояния.

## Impact

- Код: `internal/taskmanager`, `internal/config`, тесты task manager/config; возможно новый небольшой пакет для событий или уведомлений, если это упростит границы ответственности.
- Конфигурация: новые переменные для Telegram без значений по умолчанию в `.env.example`; секреты не коммитятся.
- Внешние системы: Linear остается источником статуса, Telegram Bot API используется только подписчиком уведомлений.
- Поведение: ошибка смены статуса в Linear не публикует событие; ошибка отправки Telegram-уведомления возвращается с контекстом через event dispatcher, чтобы оркестратор видел сбой реакции.
