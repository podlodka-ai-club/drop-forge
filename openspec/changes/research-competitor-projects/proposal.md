## Why

Нужно провести ресерч открытых проектов конкурентов из организации `podlodka-ai-club`, чтобы понять, какие идеи и практики стоит перенести в развитие оркестратора. Сейчас задача DRO-39 сформулирована как исследовательская, поэтому результат должен быть воспроизводимым отчетом с источниками, выводами и применимыми идеями, а не немедленным изменением runtime-кода.

## What Changes

- Добавить исследовательский артефакт по конкурентным проектам из списка репозиториев `https://github.com/orgs/podlodka-ai-club/repositories?type=all`.
- Зафиксировать методику: актуальный список репозиториев берется на момент выполнения, для каждого проекта изучаются README, структура, issues/PR и заметные архитектурные решения.
- Сформировать сравнительную матрицу по проектам: назначение, стек, orchestration-подход, интеграции, UX/CLI, observability, работа с задачами и PR, сильные идеи.
- Выделить shortlist идей для текущего оркестратора с оценкой пользы, сложности, рисков и рекомендуемого порядка внедрения.
- Не менять runtime-поведение оркестратора в рамках этого change; последующие продуктовые или технические изменения должны оформляться отдельными OpenSpec changes.

## Capabilities

### New Capabilities

- `competitor-research`: требования к воспроизводимому research-отчету по конкурентным GitHub-проектам и к формату списка идей для дальнейшего развития оркестратора.

### Modified Capabilities

- Нет.

## Impact

- Новый markdown-отчет в документации проекта, например `docs/research/competitor-projects.md`.
- Новая OpenSpec capability `competitor-research` для фиксации критериев приемки research-задач такого типа.
- Код, CLI, конфигурация `.env`, `architecture.md` и существующие runtime-компоненты не затрагиваются, если в ходе implementation не будет принято отдельное решение расширять поведение системы.
