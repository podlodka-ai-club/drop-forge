## Why

Сейчас `orchv3` создаёт PR с OpenSpec proposal, но ссылка на логи прогона остаётся только во внешней системе логирования и не попадает в сам артефакт ревью. Из-за этого при разборе проблем или проверке результата приходится отдельно искать нужный дашборд вручную.

## What Changes

- Добавить опциональную runtime-переменную `PROPOSAL_LOGS_URL` для ссылки на страницу логов proposal workflow.
- Расширить тело создаваемого PR: если `PROPOSAL_LOGS_URL` задан, runner добавляет в описание отдельный блок со ссылкой на логи.
- Сохранить текущее поведение без регрессий: если `PROPOSAL_LOGS_URL` пустой, PR создаётся как сейчас, без дополнительного блока.
- Обновить шаблон `.env.example` и пользовательскую документацию по proposal runner, чтобы новое поведение было видно и воспроизводимо локально.
- Добавить тесты на формирование тела PR и на передачу ссылки в `gh pr create`.

## Capabilities

### New Capabilities

None.

### Modified Capabilities

- `codex-proposal-pr-runner`: PR workflow получает опциональную конфигурацию ссылки на логи и публикует её в теле создаваемого PR.

## Impact

- Код: `internal/config`, `internal/proposalrunner`.
- Документация: `.env.example`, `README.md`, `docs/proposal-runner.md`.
- Внешние зависимости: новых нет; используется уже существующий `gh pr create`.
- Поведение: меняется только содержимое PR body при наличии новой переменной окружения.
