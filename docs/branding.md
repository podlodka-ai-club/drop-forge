# Брендинг Purrch

`Purrch` — публичное человекочитаемое имя проекта для README, документации, визуальной идентичности и коротких описаний продукта.

`orchv3` остается текущим техническим именем Go-модуля, CLI-бинаря, package paths, команд и runtime-конфигурации. Существующие скрипты должны продолжать вызывать CLI как `orchv3`.

Используйте `Purrch`, когда описываете проект для людей:

- заголовок README и вводное описание;
- страницы документации, которые объясняют продукт или workflow на верхнем уровне;
- визуальные ассеты, например [docs/assets/logo.svg](assets/logo.svg).

Используйте `orchv3`, когда речь идет о технических деталях:

- CLI-команды вроде `go run ./cmd/orchv3`;
- Go module, package paths, бинарь и внутренние части репозитория;
- environment variables, config keys, логи и automation scripts.

Переименование бинаря, Go-модуля, package paths, environment variables или имени репозитория — отдельная миграция и намеренно не входит в это branding-изменение.
