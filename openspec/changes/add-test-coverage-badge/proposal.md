## Why

Сейчас в репозитории нет видимого индикатора качества тестового покрытия, поэтому разработчик или ревьюер не может быстро оценить текущее состояние проверок прямо на GitHub. Для DRO-24 нужно вывести процент покрытия в README или другом заметном месте, чтобы этот сигнал был доступен без ручного запуска `go test`.

## What Changes

- Добавить в корневой `README.md` видимый процент тестового покрытия кодовой базы.
- Ввести воспроизводимый локальный способ посчитать покрытие Go-кода через стандартные команды проекта.
- Задокументировать, откуда берется отображаемый процент и как его обновлять.
- Не менять публичное поведение CLI и runtime-конфигурацию приложения.

## Capabilities

### New Capabilities

- `test-coverage-reporting`: правила расчета и публикации процента тестового покрытия для Go-кодовой базы.

### Modified Capabilities

- `project-readme`: README должен показывать процент тестового покрытия и ссылаться на способ его проверки.

## Impact

- `README.md`: появится видимый индикатор покрытия и краткая инструкция проверки.
- Скрипты, Makefile или документация разработки: может появиться команда для расчета покрытия, если текущих команд недостаточно.
- Go-тесты: поведение тестов не меняется, но в workflow разработки добавляется расчет покрытия через `go test`.
- CI/CD и внешние сервисы не обязательны для минимального решения, если процент можно надежно поддерживать в репозитории без новой интеграции.
